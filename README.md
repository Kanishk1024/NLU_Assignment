# Natural Language Understanding: Assignment 2

This repository contains the code and resources for Assignment 2, which is divided into two main problems:
1. **Problem 1:** Corpus generation and Word2Vec implementation (CBOW & Skip-gram).
2. **Problem 2:** Character-level Indian name generation using RNN variants (Vanilla RNN, BiLSTM, Attention RNN).

## Directory Structure

Ensure your project directory looks like this before running the code:

    NLUASSIGN2/
    ├── .venv/                      # Python virtual environment
    ├── requirements.txt            # List of required Python packages
    ├── problem1/
    │   ├── pdfs_academic/          # Directory containing target PDF files
    │   ├── pdfs_academic_reg/      # Directory containing target PDF files

## Prerequisites & Installation

Before running the scripts or notebooks, ensure you have the required Python libraries installed in your virtual environment. 

Run the following command in your terminal from the root directory (`NLUASSIGN2/`):

    pip install -r requirements.txt

*(Note: The `scraper.py` script will automatically download the required NLTK datasets `stopwords`, `wordnet`, and `omw-1.4` on its first run.)*

## Running Problem 1: Word2Vec

**Step 1: Generate the Corpus**
Navigate to the `problem1` directory. Ensure your `pdfs_academic` and `pdfs_academic_reg` folders are present and contain the required PDF files. Run the scraper to crawl the IITJ website and parse the PDFs/text.

    cd problem1
    python3 scraper.py

*This will create a `corpus.txt` file in the `problem1` directory containing the cleaned and lemmatized tokens.*

**Step 2: Train and Analyze Word2Vec Models**
You can run the analysis either via the terminal or interactively in Jupyter:
* **Option A (Terminal):** Run `python3 problem1.py` to execute the grid search and generate the visualizations directly.
* **Option B (Jupyter):** Open `problem1.ipynb` and run the cells sequentially. 

Both options will automatically read `corpus.txt`, display the word cloud, execute the grid search for CBOW and Skip-gram models, perform analogy tests, and plot the t-SNE visualizations.

## Running Problem 2: Character-Level RNNs

**Step 1: Generate the Dataset**
Navigate to the `problem2` directory. Run the dataset generation script to create a list of 1,000 synthetic Indian names using the Faker library.

    cd ../problem2
    python3 names.py

*This will create a `TrainingNames.txt` file in the `problem2` directory.*

**Step 2: Train Models**
You can execute the training pipeline using either the script or the notebook:
* **Option A (Terminal):** Run `python3 problem2.py`.
* **Option B (Jupyter):** Open `problem2.ipynb` and run the cells sequentially.

**Pipeline Details:**
* The code will read `TrainingNames.txt` and train the Vanilla RNN, BiLSTM, and Attention RNN models. 
* During training, it automatically tracks the lowest loss and saves the best weights to disk (`best_vanilla_model.pth`, `best_blstm_model.pth`, `best_attention_model.pth`).

**Step 3: Evaluate Saved Models**
Once the models are trained and the `.pth` files are saved in the `problem2` directory, you can run the standalone evaluation script to generate names and calculate metrics without having to retrain:

    python3 evaluation.py

This script will load the saved weights, compute the Novelty and Diversity metrics, and print a qualitative sample of 10 generated names for each architecture.

## Hardware Note
The scripts and notebooks are configured to automatically detect and use a GPU (`cuda`) if available to significantly speed up training and inference. If a GPU is not found, they will gracefully fall back to processing on the `cpu`.
